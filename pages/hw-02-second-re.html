    作業九的邏輯是有問題的喔。假設小明全年個人所得為 1100000，那它應該被課 5000 + 1100000 * 0.8 的稅收
    但依照你的程式碼邏輯
    mingBill += mingMoney; // 變數 mingBill 會被賦值為 5000 + 1100000 * 0.2
    那這樣小明等於只被課了 0.2 的稅收而已，所以有誤
  • 作業 10
    這邊可以不需要使用迴圈的，直接單純的 if, else if 判斷即可
    舉例來說，依照你的假設 playerA, playerB 都是等於 "paper"，那麼進入迴圈的前兩次判斷都會失敗，但如果不使用迴圈，直接使用 if, else if 判斷，程式碼只要由上往下跑一次即可，不會浪費到效能。
    再來，如果程式碼是用迴圈的撰寫方式，則流程圖的第一個判斷決策符號「判斷 A 是否出剪刀」就會是錯誤的了，應該會對應到「判斷 playerA 以及 playerB 是不是同時等於 state[i]」，如果是，則箭頭會指向「console.log("平手")」的處理符號。
    如果否，則箭頭會指向下一個判斷決策符號「playerA 是不是等於 "soccer"」
